Login Page with OTP

Project Overview

This website allows you to perform 2-step verification. When logging in, you will need to enter an OTP that will be sent to your email.

Key Features

  • During the sign-up process on the website, for enhanced security, the password is hashed using the "SHA-256" algorithm and stored in the database.
  • When logging in on the website, it will first check the entered password against the hashed password in the database.
  • If the passwords match, you will receive an OTP in the email you used for signing up.
  • Only after successful OTP verification will you be granted access to the system.

Login With Face

Project Overview

A secure authentication system using OpenCV and Haar Cascade Classifier for face recognition.

How It Works

  • User Registration Process: During the sign-up process, users are required to provide their details along with a photo. This photo is captured and saved in the system for future face recognition purposes.
  • Face Recognition Technology: The website employs OpenCV (cv2) and Haar Cascade Classifier for face detection and recognition. This technology ensures accurate and efficient identification of users by analyzing facial features.
  • Login Procedure: When logging in, the system captures the user's current photo and compares it against the stored photo using the face recognition algorithm. If the facial features match, the user is granted access to their portal.
  • User Portal Access: Upon successful face recognition, the user is directed to their personalized user portal, providing a seamless and secure login experience.

Insider Threat Detection using Machine Learning

Project Overview

This project is a comprehensive solution designed to detect insider threats within an organization by collecting real-time user activity logs and applying machine learning algorithms to identify anomalous behavior. Built using Flask, Socket.IO, and other technologies, the system ensures secure data transmission and effective threat detection.

How This Code Works?

  • Live Log Collection: A lightweight client application continuously collects system logs, including file access, USB events, website visits, system info, login time, and more. These logs are securely sent to the central server.
  • Machine Learning: The server uses ML models (like Isolation Forest) to analyze incoming logs and detect abnormal activities. Detected anomalies generate real-time alerts for the admin.
  • Alert Dashboard: A real-time Flask dashboard displays system activity, alerts, and user risk scores. Admin can accept or reject users, monitor behavior, and respond accordingly.
  • Secure File Sharing: A built-in secure file sharing system ensures confidential documents can be shared among users with access control mechanisms.
  • Technology Stack: Python, Flask, Socket.IO, MongoDB/SQLite, scikit-learn, HTML/CSS/JS, Bootstrap, and more.
